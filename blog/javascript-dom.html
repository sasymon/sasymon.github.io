Javascript fundamentals

What is javascipt?

If you compare HTML and CSS with the bakery window and display; it has lots of delicious things to show off, presentating an array of tempting treats for the passerby; a feast for the eyes, then javascript is the kitchen; recieves the orders, bakes the cakes, decorates the cakes, manages the ingredient stock, and methods to make the cakes. Javascript is the workhorse, the behind the scenes engine that makes everything happen.

For example with HTML and CSS we can make a pretty looking Clicks: 0 with a counter but without Javascript doesn't do anything. If we attach some javascript we can make it so every time we click this Clicks: 0 we can make the counter go up by 1. Pretty neat!

How does Javascipt work?

That's a big question, let's break it down.

Simply put, Javascript works like a set of instructions. It takes the input (variables), carries out it's instructions, and then outputs the results. Sounds simple right? Not quite, there's a lot that goes into this to get the right result, so lets get started.

Variables

Variables are basically sets of information, these can range from a set of 1 to... a lot. These variables come in five types, the first three are relative simple but the last two can get pretty complicated;

  • Strings - which are strings of text characters denoted by marks ie "Hello, this is a string"
  • Numbers - which are... numbers,
  • Boolean - which are true/false statements,
  • Arrays,
  • and Objects.
Arrays and objects need a bit more explanation.

Arrays and objects

Objects and arrays are variables that contain a collection of information rather than just one point of data like strings, numbers, and boolean variables.

The difference between arrays and objects is that arrays contain one collection of information, where Objects contain multiple collections under a key and value system (key: value). Confused? Yeah me too, so here's a few handy images to help us out. Here to the right we have a variable called chocolateGateau which is an object, inside the object is a list of the ingredients(keys) and the required amounts (values), we have two collections of variables; the ingredients, and the amount of each we need.

An important difference between arrays and objects is how they are called upon; arrays are called upon using a zero index contained with brackets- arrays start counting at , where objects use a dot notation to indicate children objects instead; each object is separated by a to denote that the next variable is a child object of the previous object.

Here we have a basic array, which can contain other different types of variables (including objects!), so to call upon 'carrot' in my arrayExample array I'd use arrayExample[2].

To call upon the chocolateFlakes key value in my object variable I'd use chocolateGateau.butterCream.chocolateFlakes stopping at the key for the value I want. And yes, you can have arrays inside of objects, and objects inside of arrays and all the way down until it's nothing but arrays and objects all the way down.

The DOM

The DOM, or Document Object Model. Everything on a website is an object which can be accessed and manipulated.

The DOM can be visually represented by a tree diagram; each object(also called a node) is a child of another object which can help us track them down for our needs.

It's good practice when using Javascript to use classes and ids to help find the specific objects you want to work with. Remember when I talked about selectors, classes and ids? No? Well then go read this.

Functions

Functions are used for code (instructions) that we want to repeat. Not only is this efficient, we have to write less code, but we can also call on it with different variables attached.

It's important to also note that declaring (writing) functions on their own won't do anything, they must be called upon to happen, this also means we can call on them multiple times.

Control flow

By default javascript runs from top to bottom in the order they were written without stopping, but fairly often the code will give instructions to change course, go back, skip or just out right stop, this is called Control flow, the sequence order of the code.

Often this achieved with if/else statements that work exactly how you'd think; if is true then do this, if not then do that.

Loops

Loops are a special action that repeat under certain circumstances, a way to manipulate control flow, for as many times as instructed, and until certain conditions are met. Loops are very versatile and come in many forms, but all have a basic starting structure.

There are mainly four types of loops;

  • for - loops through a block of code a number of times
  • for... in - loops through the propeties of an onject
  • for... of - loops through the values of the object
  • while - loops through a block of code while a specific condition is true
