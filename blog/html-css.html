What are the best practices associated with using classes vs. ids?

What are classes and ids

Classes and ids are selectors. Selectors help define elements (contents of a webpage); think of them as containers that hold content and help tell them what to do.

Class and id selectors define a ruleset under a given value, usually in an external stylesheet. We can then give these rulesets to elements on our webpage by attributing the given value to elements. For example we can turn this text red, or assign it a different font.

Classes are defined in the stylesheet by a followed by a value, and ids by a followed by a value.

What's the difference between classes and ids?

Classes and ids function in much the same manner, the difference lies in their application:

  • id selectors only can be used on ONE element on a webpage.
  • Class selectors can be used on any number of elements.

Elements are also able to have multiple of either classes or ids, when this is the case and there's conflicting instructions the order in which they are written in the stylesheet matters, the last one takes precedence.

When an element has both a class and an id, the id will always take precedence when there are conflicting attritbutes. This allows for what's called specificity; you can get really specific when you apply multiple different classes and ids to the same element!

When to use a class or an id?

It's generally good practice to use classes when you're wanting to style multiple elements, and id when you're styling a specific element. Addtionally, because ids are unique on a webpage they can also be referred to within code, for exmaple using it in a hyperlink.
